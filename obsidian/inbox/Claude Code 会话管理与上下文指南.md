## 核心概念

### Context Rot（上下文腐化）
随着上下文不断增长，模型的表现会逐渐下降——注意力被分散到更多 token 上，较早、较弱相关的内容开始干扰当前任务。

### Compaction（压缩）
接近 100 万 token 上限时自动触发，也可手动执行 `/compact`。压缩会用摘要替换原始历史，存在信息损失，但 Claude 会尽量保留重要结论。

---

## 5 种上下文管理手段

| 场景 | 工具 | 原因 |
|------|------|------|
| 同一任务继续推进，上下文仍相关 | **Continue** | 窗口内容仍有效，无需重建 |
| Claude 走错方向 | **Rewind**（双击 Esc / `/rewind`） | 保留有用的文件读取，丢掉失败尝试，结合新认识重新提示 |
| 会话臃肿但任务未完 | **/compact \<hint\>** | 成本低，Claude 判断重要内容；hint 可引导压缩方向 |
| 开始全新任务 | **/clear** | 上下文最干净；由自己决定哪些信息带过去 |
| 下一步产生大量中间输出，只需结论 | **Subagent** | 中间噪音留在子代理上下文，主会话只接收最终结果 |

## 实用技巧

### Rewind 优于事后纠正
与其输入"这个方法不行，试试 X"，不如回退到文件读取完成的时间点，直接提示"不要用 A 方案，走 B 方案"。

### 引导压缩方向
```
/compact focus on the auth refactor, drop the test debugging
```
手动触发时加 hint，避免压缩丢掉关键信息。

### 避免糟糕的自动压缩
糟糕的压缩通常发生在**调试过程中触发**——摘要围绕排障展开，而你下一步要转向别的任务，关键信息已丢失。

解决方案：在任务切换前主动 `/compact`，并说明接下来的方向。

### Subagent 使用场景
```
启动一个 subagent，根据下面这份规范文件验证这项工作的结果。
分出一个 subagent 去阅读另一个代码库，总结它是如何实现认证流程的。
分出一个 subagent，根据我的 git 改动来撰写这个功能的文档。
```
