# frontend-design

创建独特、生产级前端界面的技能，注重高设计质量。

## 功能描述

当用户要求构建 Web 组件、页面、应用程序时激活，包括：
- 网站
- 落地页
- 仪表板
- React 组件
- HTML/CSS 布局
- 任何 Web UI 的样式/美化

## 设计思维

编码前理解上下文并确定大胆的美学方向:
- **目的**: 这个界面解决什么问题？谁使用它？
- **基调**: 选择一个极端：粗野极简、极繁混乱、复古未来主义、有机自然、奢华精致、玩具般有趣、编辑杂志风、粗野原始、装饰艺术几何、柔和粉彩、工业实用等
- **约束**: 技术要求（框架、性能、可访问性）
- **差异化**: 是什么让它令人难忘？

## 美学指南

### 排版

选择美丽、独特、有趣的字体:
- 避免 Arial 和 Inter 等通用字体
- 选择能提升前端美学的独特选择
- 配对独特的展示字体与精致的正文字体

### 颜色与主题

致力于有凝聚力的美学:
- 使用 CSS 变量保持一致性
- 主色与锐利强调色优于胆小、均匀分布的调色板

### 动画

使用动画产生效果和微交互:
- 优先使用纯 CSS 解决方案
- 使用 Motion 库处理 React
- 专注于高影响力时刻
- 使用滚动触发和悬停状态带来惊喜

### 空间构图

- 出乎意料的布局
- 不对称
- 重叠
- 对角流动
- 打破网格的元素
- 大量留白或受控密度

### 背景与视觉细节

创造氛围和深度:
- 渐变网格
- 噪点纹理
- 几何图案
- 分层透明度
- 戏剧性阴影
- 装饰边框
- 自定义光标
- 颗粒叠加

## 绝不使用

- 通用字体（Arial、Inter、Roboto、系统字体）
- 常见的颜色方案（特别是白色背景上的紫色渐变）
- 可预测的布局和组件模式
- 缺乏上下文特定个性的千篇一律设计

## 重要提示

- 匹配实现复杂度与美学愿景
- 极繁设计需要精心策划的代码与广泛动画和效果
- 极简或精致设计需要克制、精确和对间距、排版和微妙细节的仔细关注
- 优雅来自良好执行愿景
