## 开发流程类

### systematic-debugging — 系统化排查 Bug

**触发时机**：遇到任何 bug、测试失败、异常行为时，**在提出修复方案之前**自动触发。

**铁律**：先找根因，再提修复。没有根因分析，不允许直接改代码。

**流程**：
1. 重现问题，确认症状
2. 列出所有可能的原因
3. 逐一验证，锁定根因
4. 根因确认后，才开始修复

```
场景：iot-min 的 AGV 任务队列消费异常
→ 先 /systematic-debugging 分析日志和调用链，再动代码
```

---

### test-driven-development — 测试驱动开发

**触发时机**：实现任何新功能或 bug fix **之前**。

**流程**：先写测试 → 跑测试看它失败 → 写最少代码让测试通过 → 重构。

**例外**：临时原型、配置文件、生成代码可跳过。

```
场景：给 open-api 的接口新增参数校验逻辑
→ 先写 JUnit 5 测试用例，跑失败后再写实现
```

---

### using-git-worktrees — Git 工作树隔离

**触发时机**：开始一个需要隔离的功能分支，或执行实现计划前。

**作用**：在同一个仓库创建多个独立工作目录，不同功能并行开发互不影响。

**优先目录顺序**：`.worktrees/` > `worktrees/` > 自定义。

```
场景：同时维护 hk 的 hotfix 和新功能开发
→ /using-git-worktrees 创建隔离工作区
```

---

### review-implementing — 实现代码评审意见

**触发时机**：收到 PR 评审意见，需要逐条落实时。

**流程**：解析评审意见 → 建 Todo 列表 → 逐条实现 → 验证。

```
场景：架构评审后有一批修改意见
→ 把评审意见粘贴给 Claude，/review-implementing 逐条处理
```

## 安全类

### VibeSec-Skill — 安全编码审查

**触发时机**：开发 Web 接口、做安全扫描或审计时自动触发。

**覆盖**：访问控制、输入校验、SQL 注入、XSS、认证授权等。

**核心原则**：纵深防御、最小权限、失败时拒绝访问。

```
场景：lot / open-api 对外接口开发
→ 自动检查接口是否有权限校验漏洞
```

---

### owasp-security — OWASP 安全标准

**触发时机**：代码审查、实现认证授权、处理用户输入时。

**覆盖**：OWASP Top 10:2025 + ASVS 5.0 + Agentic AI 安全（2026）。

**快速参考**：

| 风险 | 防御要点 |
|------|---------|
| 访问控制失效 | 默认拒绝，服务端校验 |
| 注入 | 参数化查询，输入验证 |
| 认证失败 | MFA，安全会话 |
| 加密失败 | TLS 1.2+，AES-256-GCM |
| 日志缺失 | 记录安全事件，结构化日志 |

```
场景：架构评审 datacenter 的接口安全性
→ /owasp-security 过一遍 Top 10 清单
```

## 设计与规划类

### brainstorming — 方案发散与设计

**触发时机**：任何创造性工作（新功能、架构设计、方案选型）**之前**必须触发。

**硬性规定**：必须先给出设计方案并获得确认，才能开始写代码。

**流程**：了解项目上下文 → 逐一提问 → 提出 2-3 个方案（含利弊）→ 确认后写设计文档。

```
场景：设计 iot-min 新的机器人类型接入方案
→ /brainstorming 先梳理需求和方案，评审通过再实现
```

---

### kanban-skill — Markdown 看板

**触发时机**：管理任务、跟踪工作项时。

**存储**：任务以 `.md` 文件存放在项目 `kanban/` 目录，无需数据库。

**状态**：`backlog` → `todo` → `doing` → `done` → `archive`

**使用方式**：直接描述需要，Claude 自动创建/移动/查看卡片。

```
场景：跟踪 yuncang AI管家的迭代任务
→ "帮我创建一个看板卡片：实现酒店入住接口"
```

## 效率工具类

### internal-comms — 内部沟通文档

**触发时机**：写内部沟通材料时自动触发。

**支持类型**：
- **3P 周报**（Progress/Plans/Problems）
- 领导汇报
- 事故复盘报告
- 项目进度更新
- FAQ

```
场景：周五写本周技术进展周报
→ "帮我写一份 3P 周报，本周完成了…"
```

---

### deep-research — 深度技术调研

**触发时机**：需要技术选型、竞品分析、文献调研时。

**前置条件**：需要配置 `GEMINI_API_KEY`（Google AI Studio 申请）。

**耗时**：2-10 分钟，生成带引用的详细报告。**费用约 $2-5/次**。

```
场景：调研是否引入 BifroMQ 替换现有 MQTT Broker
→ /deep-research "对比 BifroMQ 和 EMQX 在 IoT 场景下的性能和运维成本"
```

## 技能触发方式

| 方式 | 说明 |
|------|------|
| **自动触发** | Claude 根据上下文自动识别并调用（如遇到 bug 自动用 systematic-debugging） |
| **手动触发** | 在对话中输入 `/技能名` 显式调用 |
| **描述触发** | 说明你要做的事，Claude 判断并提示是否调用 |
