## 3. 社交媒体运营

### 🔴 小红书 (XiaoHongShu)

| 技能 | 用途 | 触发词 |
|------|------|--------|
| `xhs-auth` | 账号认证管理 | "登录小红书"、"检查登录状态"、"切换账号" |
| `xhs-explore` | 内容发现与搜索 | "搜索小红书"、"查看笔记详情"、"浏览首页" |
| `xhs-interact` | 社交互动 | "评论"、"回复"、"点赞"、"收藏" |
| `xhs-publish` | 内容发布 | "发布内容"、"上传图文"、"发视频"、"定时发布" |
| `xhs-content-ops` | 复合运营工作流 | "竞品分析"、"热点追踪"、"内容创作" |

**使用流程：**
```
1. 登录账号 → xhs-auth
2. 发现内容 → xhs-explore
3. 互动操作 → xhs-interact
4. 发布内容 → xhs-publish
5. 复合任务 → xhs-content-ops
```

## 7. 开发与测试

### 🧪 测试

| 技能 | 用途 | 触发词 |
|------|------|--------|
| `qa` | Web 应用 QA 测试 | "QA 测试"、"找 bug"、"测试 X 功能" |
| `test-driven-development` | TDD 测试驱动开发 | "用 TDD"、"先写测试" |
| `gstack` | Headless 浏览器测试 | "浏览器测试"、"截图验证" |
| `webapp-testing` | 本地应用测试 | "测试本地应用"、"Playwright" |

**QA 模式选择：**

| 模式 | 触发条件 | 适用场景 |
|------|----------|----------|
| `diff-aware` | feature 分支 | 自动分析 git diff，测试受影响页面 |
| `full` | 完整回归 | 系统性全面探索 |
| `quick` | 快速冒烟 | 30 秒快速检查 |
| `regression` | 对比基线 | 与之前版本对比 |

---

### 🔍 调试

| 技能 | 用途 | 触发词 |
|------|------|--------|
| `systematic-debugging` | 系统化调试方法 | "调试 X"、"X 不工作"、"修复 bug" |

**调试工作流：**
```
1. 遇到 bug → systematic-debugging 激活
2. 分析问题 → 不要盲目修复
3. 定位根因 → 系统化排查
4. 验证修复 → 确保不引入新问题
```
