# qa

系统化 QA 测试技能，生成带证据的详细报告。

## 功能描述

用于 "qa"、"QA"、"测试此站点"、"查找 bug" 或审查质量。

## 模式

| 模式 | 描述 |
|------|------|
| diff-aware | 在功能分支自动 - 分析 git diff |
| full | 系统化探索（默认） |
| quick | 30 秒冒烟测试 |
| regression | 与基线对比 |

## 工作流程

### Phase 1: 初始化

1. 找到浏览二进制文件
2. 创建输出目录
3. 复制报告模板

### Phase 2: 认证（如需要）

```bash
$B goto <login-url>
$B snapshot -i
$B fill @e3 "user@example.com"
$B fill @e4 "[REDACTED]"
$B click @e5
$B snapshot -D
```

### Phase 3: 定向

```bash
$B goto <target-url>
$B snapshot -i -a -o "$REPORT_DIR/screenshots/initial.png"
$B links
$B console --errors
```

### Phase 4: 探索

每页检查:
1. **视觉扫描** - 布局问题
2. **交互元素** - 按钮、链接、控制是否工作
3. **表单** - 填写和提交
4. **导航** - 所有路径
5. **状态** - 空状态、加载、错误、溢出
6. **控制台** - 交互后新的 JS 错误
7. **响应式** - 移动端视口

### Phase 5: 记录

**交互 bug**（带证据）:
1. 操作前截图
2. 执行操作
3. 结果截图
4. 用 `snapshot -D` 显示变化
5. 写引用截图的复现步骤

**静态 bug**:
1. 单个带注释截图显示问题
2. 描述问题

### Phase 6: 总结

1. 计算健康分数
2. 写 "Top 3 修复项"
3. 写控制台健康摘要
4. 更新严重性计数
5. 填写报告元数据
6. 保存基线 JSON

## 健康分数计算

### 控制台 (权重: 15%)
- 0 错误 → 100
- 1-3 错误 → 70
- 4-10 错误 → 40
- 10+ 错误 → 10

### 链接 (权重: 10%)
- 0 断开 → 100
- 每个断开链接 → -15

### 每类别扣分
- Critical → -25
- High → -15
- Medium → -8
- Low → -3

## 重要规则

1. **复现是关键** - 每个问题至少一张截图
2. **记录前验证** - 重试确认问题
3. **绝不包含凭证** - 密码写 `[REDACTED]`
4. **逐步记录** - 边发现边记录
5. **不读源码** - 以用户身份测试
6. **每次交互后检查控制台** - JS 错误仍可能是 bug
7. **像用户一样测试** - 使用真实数据
8. **深度优于广度** - 5-10 个有证据的详细问题 > 20 个模糊描述
