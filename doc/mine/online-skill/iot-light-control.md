## 任务目标

用户表达了开灯或关灯需求，你需要：
1. 从参数或上下文中提取 `roomNumber`（房间号）和 `action`（开灯 / 关灯）
2. 执行开关灯指令
3. 返回操作结果给用户

## 第一步：提取参数

**roomNumber（房间号）** 按以下优先级获取：
1. `/iot-light-control <roomNumber> <action>` 命令行参数 `$ARGUMENTS` 第一个值
2. 对话上下文中提到的房间号（如"301"、"三零一"、"3楼01号"）

**action（操作）** 按以下优先级获取：
1. `/iot-light-control <roomNumber> <action>` 命令行参数 `$ARGUMENTS` 第二个值（`on` 或 `off`）
2. 从用户语义推断：
   - "开灯"、"打开"、"亮灯"、"开一下灯" → `on`
   - "关灯"、"关掉"、"熄灯"、"关一下灯" → `off`

**roomNumber 为必传**。如果无法从上下文获取，告知用户提供房间号，不要猜测或伪造。

**action 为必传**。如果意图不明确，询问用户是开灯还是关灯。

## 第二步：执行

不发送任何 HTTP 请求，直接根据参数判断响应：
- `roomNumber` 非空 + `action` 为 `on` 或 `off` → 成功
- `roomNumber` 为空 → 失败，提示房间号缺失
- `action` 不合法 → 失败，提示操作类型无效

## 第三步：输出结果

**成功时：**

```
✅ 操作成功

房间：{roomNumber}
操作：{on → 已开灯 / off → 已关灯}
时间：{当前时间}
```

**失败时：**

```
❌ 操作失败

原因：{失败原因}
```

## 约束

- 不发送任何真实 HTTP 请求
- 不自动猜测房间号，必须由用户提供或从上下文明确获取
- 操作完成后不自动触发其他 skill
