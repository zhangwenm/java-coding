## 接口配置

- 统一接口地址：`http://localhost:3430/l4/v1/skills/execute`

## 任务目标

用户表达了喝水/点位查询需求，你需要：
1. 从当前对话上下文中提取 `storeId`
2. 调用统一接口获取门店所有点位
3. （可选）根据用户提到的点位名称/编号进行匹配
4. 将结果整理后呈现给用户，供其选择

## 第一步：提取参数

**storeId** 按以下优先级获取：
1. `/hdos-get-points <storeId> [point]` 命令行参数 `$ARGUMENTS` 第一个值
2. 对话上下文中的 `storeId`

**point**（可选）按以下优先级获取：
1. `/hdos-get-points <storeId> <point>` 命令行参数 `$ARGUMENTS` 第二个值
2. 对话上下文中用户提到的点位编号/名称（如"308"、"101房间"、"充电桩"）

`storeId` 为必传。如果无法获取，告知用户提供，不要猜测或伪造。
`point` 为可选——不传则展示全部点位列表。

## 第二步：调用统一接口

接口入参仅需 `storeId`，不再需要 `point`/`pointType` 等服务端过滤参数。

```bash
curl -s -X POST "http://101.42.172.33:3430/l4/v1/skills/execute" \
  -H "Content-Type: application/json" \
  -d '{
    "skillId": "20260415160436877_MD_hdos-get-points",
    "skillParameters": {
      "storeId": "{storeId}"
    }
  }'
```

## 第三步：处理响应

**响应结构**：
```json
{
  "code": 200,
  "message": "success",
  "traceId": "...",
  "data": [
    {
      "num": 1,
      "name": "101",
      "type": "0",
      "floor": -1,
      "avatar": "默认",
      "mapName": "dongshengfuyilou",
      "placeId": "9b691ff86cf447adaac056d233563923",
      "version": "d23d28a5f7a94b039dd2982dfb2b2dc6",
      "properties": ""
    }
  ]
}
```

**失败时**：
- `code !== 200`：展示 `message` 字段原始内容

**成功（code=200）时**：

### 点位类型说明

| type | 含义 |
|------|------|
| `"0"` | 普通点位（桌号/房间号） |
| `"11"` | 充电桩 |

### 楼层展示

`floor` 为数字。展示规则：
- `-1` → `B1`
- `-2` → `B2`
- `0` → `1楼`（或根据业务约定）
- `1` → `1楼`
- 以此类推

### 匹配流程（用户指定了 point 时）

**第一步：精准匹配**

用 `point` 与每条点位的 `name` 字段做完全相等比较（忽略大小写、全半角）。

- 匹配成功 → 输出该点位，标注 `✅ 精准匹配`，流程结束

**第二步：模糊匹配（精准未命中时）**

**分词与归一化：**

从 `point` 中提取以下 token：
- 连续数字串（如 "1001房间" → `["1001"]`）
- 有效汉字词（去除无意义词：的、号、室、层、楼、房、间、个）
- 保留方位词（东南西北前后左右）和类型词（充电桩、会议室、餐厅、大堂、前台、包厢等）

**数字 token 的扩展匹配规则（按优先级依次尝试）：**

1. **包含匹配**：点位 `name` 包含该数字 token，或该数字 token 包含点位 `name`（双向子串）
2. **编辑距离匹配**：数字 token 与点位 `name` 的编辑距离 ≤ 1（如 "1001" 匹配 "101"）；若数字长度 ≥ 5，允许编辑距离 ≤ 2
3. **前缀/后缀匹配**：点位 `name` 是数字 token 的前缀或后缀

对所有点位按以下评分排序，取最高分：
- 精准子串命中：3分
- 编辑距离=1：2分
- 编辑距离=2：1分
- 非数字 token 命中（type 类型匹配等）：每个 +1分

分数相同时取 `name` 更短者。

- 找到候选 → 输出该点位，标注 `🔍 模糊匹配（您输入："${point}"）`，并询问用户是否确认
- 完全无任何候选 → 告知用户：`未找到与"${point}"匹配的点位，请检查输入或从以下列表选择：`，然后按楼层分组展示全部普通点位（type="0"）

### 点位展示格式

**单个匹配点位：**
```
✅ 精准匹配 / 🔍 模糊匹配（原始输入："308"）

点位：308
楼层：3楼
类型：普通点位
```

**列表模式（未指定 point 或匹配失败时），按楼层分组：**
```
📍 B1
  · 101 — 普通点位
  · 102 — 普通点位
  · 103 — 普通点位
  · charge_point_-1F_110104630 — 充电桩 🔋
  · charge_point_-1F_110102895 — 充电桩 🔋
```

充电桩点位在 `properties` 中包含 `charging_pile_type`（充电桩类型）和 `cabin_key`/`chassis_key`。展示时若有 `chassis_key`，可补充展示底盘编号。

## 约束

- 不要在没有真实 `storeId` 的情况下发起请求
- 不要自动帮用户选择点位，等待用户回复后再选择
- 不要自动推进到下单步骤
- 充电桩（type="11"）点位仅当用户明确需要充电服务时才推荐，默认只展示普通点位（type="0"）
